// The Problem

Why Students Waste Hours Every Week

Before UniPortal, academic information was scattered across multiple disconnected systems.

  • 🌐

    Information Overload & Dispersal

    Students manually check notice boards, faculty pages, department emails, and course portals — all separately.

  • 🗓️

    Static & Outdated Data

    Timetables and syllabus PDFs uploaded once, never updated when changes happen mid-semester.

  • 🔒

    No Role Differentiation

    Faculty needed separate admin access; students saw too much or too little. No unified access model existed.

Our Solution Architecture

1
Faculty & Admin upload data via secure portal
2
Express.js REST API validates JWT and routes requests
3
MongoDB stores structured course, faculty & schedule data
4
React frontend fetches and renders updates in real time
5
Students see clean, filtered, role-specific dashboards
